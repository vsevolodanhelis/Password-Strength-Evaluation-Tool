\chapter{Glossary of Key Terms}
\begin{description}
    \item[Entropy] A measure of unpredictability or information content, measured in bits. Higher entropy means greater resistance to brute-force attacks.
    \item[Bit (of entropy)] A unit of information representing a binary choice. Each bit doubles the number of possible passwords.
    \item[log₂] Logarithm base 2. Used in entropy calculation to determine how many bits are needed to represent the password space.
    \item[Levenshtein distance] The minimum number of single-character edits (insertions, deletions, substitutions) required to change one word into another.
    \item[Brute-force attack] An attack that tries every possible combination until the correct password is found.
    \item[Dictionary attack] An attack that tries words from a pre-defined list of common passwords.
    \item[Credential stuffing] Using username/password pairs from one breach to attempt logins on other services.
    \item[Pattern attack] An attack that exploits common human patterns in password creation (e.g., ``qwerty'', ``123456'', repeated characters).
    \item[Leetspeak] A system of modified spellings using numbers and symbols to replace letters (e.g., ``@'' for ``a'', ``3'' for ``e'').
    \item[Crack time] Estimated time to guess a password via brute-force, calculated as $2^{entropy} / (guesses per second)$.
    \item[False negative] A weak password incorrectly rated as strong.
    \item[False positive] A strong password incorrectly rated as weak.
    \item[Static charset] A fixed set of characters used in entropy calculation (less accurate than dynamic charset).
    \item[Dynamic charset] A character set that adapts based on which character types actually appear in the password.
    \item[Scoring constants] Predefined values that determine how much each factor (length, diversity, etc.) contributes to the final score.
    \item[Heuristic] A problem-solving approach based on experience and intuition rather than rigorous mathematical proof.
    \item[Threat model] A specification of the types of attacks a security mechanism is designed to defend against.
    \item[Offline attack] An attack where the attacker has obtained a cryptographic hash and attempts to crack it without interacting with the target system.
    \item[Online attack] An attack that requires interaction with a target service (e.g., web login form) and is subject to rate limiting.
    \item[Passphrase] A sequence of words used for authentication (e.g., ``correct horse battery staple'').
    \item[Concept drift] The change in what constitutes a ``common'' password over time, requiring periodic updates to password blacklists.
\end{description}
